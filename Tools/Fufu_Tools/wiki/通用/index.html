芙芙工具箱通用问题(Q&A)

适用于芙芙工具箱 for windows

 

Q：如何找到芙芙工具箱的安装路径

A：

你可以运行芙芙工具箱的信息确认工具来查找芙芙工具箱的安装路径。

请按照以下步骤操作：

    1. 打开芙芙工具箱

  • 2. 打开关于芙芙工具箱 > 信息确认

  • 3. 在信息确认工具的--- Directory where Fufu Tools is located ---处即可看到芙芙工具箱的安装位置

Q：我可否使用仓库内的工具而非发行版的工具？这样做是否也受支持？

A：

可以的，不过请在使用前执行git pull更新一下仓库，以解决一些可能的问题。

这样做相当于使用code版程序。详细版本分类请查看[芙芙工具箱版本说明]

我仅授予用户仓库文件的使用权，请勿将文件进行二次分发、上传至其他仓库等，详情请查看LICENSE 文件

对于这样做是否也受支持，请查看SECURITY 文件

对于下载仓库文件zip，不属于使用code版程序，这样做不受支持。

 

Q：我该如何更新芙芙工具箱

A：

在发布新版本后，你可以更新芙芙工具箱。

目前由于技术原因，更新程序尚未制作，只制作了更新信息检测程序。所以目前如果需要更新芙芙工具箱请先卸载/删除本地的旧版本，然后安装/下载新版本。

请注意，如果需要保存您之前的配置，在卸载旧版前请先另存旧版中的除初始目录中的config.ini外所有的config.ini文件。详细信息请参阅[Q:我该如何在更新时保留我的配置?]

如果您是使用code版芙芙工具箱，请cd到程序目录后运行git pull拉取最新源码

 

Q：我该如何在更新时保留我的配置？

A:

目前由于技术原因，暂未制作更新程序。如果直接删除/卸载旧版本后再下载/安装新版本，则会丢失旧版本中的所有配置

如果您希望保存您的配置并将其用于新版本，目前的办法是先另存旧版本中的除初始目录中的所有config.ini文件，在更新完成后用这些config.ini文件替换新版本中的config.ini文件。

请注意以下几点：

  1. 在另存旧版config.ini文件时请勿也另存初始目录中的config.ini文件。

  2. 在替换新版的config.ini文件前，请先阅读新版本的更新说明，切勿替换更新说明中提到的config.ini文件。如果你这样做，你可能会遇到一些错误。

  3. 切勿将旧版config.ini文件另存在初始目录中，以防旧版config.ini文件被误删。

Note:初始目录指的是在[Q：如何找到芙芙工具箱的安装路径]中程序输出的目录，也就是芙芙工具箱的目录。

自v1.3.0起，你可以使用以下方法另存配置文件:

  1. 打开芙芙工具箱
  2. 选择关于芙芙工具箱>另存配置文件
  3. 在新窗口中选择需要另存到的文件夹，切勿将配置文件另存在旧程序的文件夹内，请确保目标文件夹为空。

Q：我可否只需要其中的一个/几个工具？

A:

可以的

程序的目录很简答，你可以直接删除Tools文件夹下你不需要的工具，删除后仍可正常使用你需要的工具。

如果你把主程序删了，你也可以运行工具，就是需要自己去找对应文件而已。
